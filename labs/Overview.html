Overview:

The AppDynamics Fundamentals reservation based Lab provides you with an environment to deploy several AppDynamic's agents along with a sample application to understand how AppDynamics provides end-to-end application monitoring and allows you to quickly resolve performance issues before they become business problems.

Lab Information:

The AppDynamics Fundamentals Lab environment consists of two CentOS Linux based VM's. The first VM hosts the AppDynamics Controller and will be referred to from this point on as the "Controller VM". The second VM hosts the Supercar-Trader sample application used in the Lab. The second VM will be the host where you will install the AppDynamics agents and will be referred to from this point on as the "Application VM".

You can reserve this lab and remotely access to install the AppDynamics agents with the sample application. After installing the AppDynamics agents with the sample application, you will access the AppDynamics Controller to monitor the application performance and troubleshoot issues in the application that are impacting end user experience.

The topology diagram for the Lab environment can be found here.

Important Note:

  • You will need to complete the Java Application Monitoring Fundamentals lab before you start the other labs in the learning track.
  • You must use the same two VM instances for each lab within the learning track as they build on one another.
  • Ensure that you increase your reservation duration when starting your sandbox reservation so you have adequate time to complete all of the labs.
  • You can increase your reservation duration when starting your sandbox reservation as described in step 4 here.
  • If your reservation expires before you complete the labs, all your changes will be lost. You can request a reservation extension before it expires using the sandbox support forum found here.

      • credentials [admin/welcome1]

  • Application VM
    • SSH - ssh centos@10.10.20.10
